NAME

DLink::Mgmt - интерфейс для управления коммутаторами D-Link

Вверх

SYNOPSIS

Модуль DLink::Mgmt дает возможность представить коммутатор в качестве объекта. Для управления используются протоколы SNMP и Telnet. На данный момент поддерживаются следующие модели:

DES-3526
DES-3528
DES-3028
DES-1228/ME/B1A
DES-3200/A1 Series
DES-3200/C1 Series
DGS-3120-24SC/A1
DGS-3100-24TG

Также поддерживается ограниченное чтение настроек на коммутаторах:

DGS-3627G
DGS-3620-28SC

Вверх

DESCRIPTION

Constants

В модуле используются следующие строковые константы:

enabled, disabled, partially, other, unknown, auto, 10-half, 10-full, 100-half, 100-full, 1000-full, 10000-full, link-pass, link-fail, none, empty, lbd, ddm, storm, local0, local1, local2, local3, local4, local5, local6, local7, all, warn, info, emergency, alert, critical, error, notice, debug, strict, loose, drop, shutdown, normal, error, loop, permit, deny

Также по умолчанию приняты следующие соглашения:

ISM VLAN tag - 24
SNMP version для readonly доступа - 2c
SNMP community для readonly доступа - dlread
Telnet login - admin
Telnet password - Masterok

Functions

new

Создает экземпляр класса DLink::Mgmt.

my $dlink = DLink::Mgmt->new(
host => 172.16.131.115,
version => '2c',
community => 'dlread',
mvr => 24,
gw => '00-25-90-91-32-9B'
);

host - адрес коммутатора, обязательный параметр.
version - версия SNMP для ReadOnly доступа.
community - SNMP community для ReadOnly доступа.
mvr - IGMP Snooping Multicast VLAN Tag.
gw - MAC-адрес шлюза сети управления.

При вызове функции new экземпляр класса DLink::Mgmt получает свойство-объект snmp, которое является экземпляром класса Net::SNMP и используется для ReadOnly вызовов. Также, если устройство не является L3-коммутатором, создается свойство-объект snmpv3, являющееся экземпляром класса Net::SNMP и использующееся для Write вызовов. Параметры для установления SNMPv3-соединения описываются в конце функции и могут различаться в зависимости от устройства. Подробнее это будет описано в разделе "Добавление новых устройств".

name

Возвращает имя модели устройства.

print $dlink->name;# DES-3526

_snmpget

Возвращает результат snmpget-запроса по указанному OID. Если OID не существует, возвращает noSuchObject.

$var = $dlink->_snmpget('1.3.6.1.2.1.1.2');
print $var; # iso.3.6.1.4.1.171.10.64.1

Одна из немногих внутренних функций, которые безопасно использовать напрямую.

_telnet_cmd

Выполняет команду в командной строке коммутатора и возвращает результат в виде списка строк. Безопасна при использовании напрямую.

model

Возвращает OID модели коммутатора.

print $dlink->model;# 1.3.6.1.4.1.171.10.64.1

error

Возвращает строку с ошибкой SNMPv3. Как правило используется в связке с _snmpset.

description

Возвращает строку с описанием коммутатора.

print $dlink->description;# DES-3526 Fast-Ethernet Switch

ip_to_hex

Возвращает шестнадцатиричное представление IP-адреса.

print DLink::Mgmt::ip_to_hex('172.16.128.200');# AC1080C8

units

Возвращает количество юнитов в стеке. Для нестекируемых коммутаторов возвращает 1.

createMask

Возвращает битовую маску на основе указанных портов.

print $dlink->createMask('1-3,9-12');# E0F0000000000000

На данный момент интервалы с указанием номера юнита не поддерживаются!

addMasks

Возвращает результат сложения двух заданных масок.

print $dlink->addMasks('00ff000000000000', 'ac00ed0000000000'); # ACFFED0000000000

subtractMasks

Возвращает результат вычитания второй указанной маски из первой.

print $dlink->subtractMasks('FFFFFFC000000000', '8000000000000000'); # 7FFFFFC000000000

AddToMask

Возвращает результат добавления в указанную маску указанных портов.

print DLink::Mgmt::AddToMask('FFFFFF0000000000', '25-26'); # FFFFFFC000000000

RemoveFromMask

Возвращает результат вычитания из указанной маски указанных портов.

print DLink::Mgmt::RemoveFromMask('FFFFFFC000000000', '1-4,20-26'); # 0FFFE00000000000

_snmpwalk

Возвращает результат snmpwalk-запроса по указанному OID в виде списка хэшей. Если OID не указан, возвращает noSuchObject.

_snmpgetnext

Возвращает результат snmpgetnext-запроса по указанному OID. Если OID не указан, возвращает noSuchObject.

_snmpset

Возвращает результат snmpset-запроса по списку параметров. Является оберткой для Net::SNMP->set_request. В случае, если запрос прошел неудачно, возвращает undef, при этом ошибку можно получить, вызвав функцию snmpv3, функция

save

Производит сохранение конфигурации и/или системного журнала коммутатора в зависимости от указанного параметра:

cfg - сохранить конфигурацию
log - сохранить системный журнал
all - сохранить все

При вызове без параметра считается, что был назначен параметр all.
Внимание! В этой функции используются OID, не указанные в конструкторе new. Подробнее в разделе "Добавление новых устройств".

reboot

Производит перезагрузку коммутатора.
Внимание! В этой функции используются OID, не указанные в конструкторе new. Подробнее в разделе "Добавление новых устройств".

uplink

Возвращает номер uplink-порта.

Важно! Функция запрашивает коммутатор по SNMP только в первый раз, все последующие вызовы возвращают значение свойства uplink_port в целях минимизации количества запросов к устройству. Поэтому в случаях, когда uplink-порт мог измениться, перед новым вызовом uplink стоит назначать свойству uplink_port значение undef.

firmwareBoot

Возвращает версию прошивки, на которой работает коммутатор в данный момент.

print $dlink->firmwareBoot;# 6.20.B07

firmwareList

Возвращает список прошивок на устройстве. Кстати, прошивка, использующаяся при загрузке, помечена символом *

@fw_list = $dlink->firmwareList;
foreach my $fw (@fw_list) {
print $fw, "\n";
}

6.00.B057
 *6.20.B018

isPortInMask

Функция проверяет, входит ли указанный порт в указанную битовую маску, и возвращает битовую маску порта. Если порт не входит в указанную битовую маску порта, возвращает 0.

print DLink::Mgmt::isPortInMask(1, 'FF00000000000000'); # 8000000000000000
print DLink::Mgmt::isPortInMask(10, 'FF00000000000000'); # 0

getPort...

Список функций, возвращающих ту или иную информацию о состоянии и настройках порта. Все нижеописанные функции реализованы в виде интерфейсов к функции _getPort, которая поддерживает стеки коммутаторов. Первым параметром должен быть указан порт, вторым (необязательным) параметром может быть указан тип порта: copper, fiber, all. Если второй параметр будет опущен, будет выбран тип порта по умолчанию (для каждой модели указан в конструкторе new). Если вторым параметром будет указано all, функция вернет список с результатами для каждого из интерфейсов, причем первым будет медный интерфейс.

getPortOperNway

Состояние согласования скорости/дуплекса на порту (link-fail, 10-half, 10-full, 100-half, 100-full, 1000-full...).

getPortOperState

Состояние порта (link-fail, link-pass).

getPortAdminNway

Административное состояние согласования скорости/дуплекса на порту (auto, 10-half, 10-full, 100-half, 100-full, 1000-full...).

getPortAdminState

Адиминстративное состояние порта (enabled, disabled).

getPortErrDisabled

Состояние Err-Disabled на порту (none, storm, lbd, error, unknown). На большинстве моделей функция не поддерживается, т.к. соответствующий OID в дереве создается только при переходе порта в состояние Err-Disabled. Если функция будет вызвана для устройства с вышеуказанным поведением, а порт будет функционировать нормально, будет возвращена строка not_supported.

getPortDescription

Возвращает пользовательское описание порта (комментарий).

getLBDState

Возвращает общее состояние функционала Loopback Detection (enabled, disabled).

getLBDPortState

Возвращает состояние настройки LBD на порту (enabled, disabled) или not_supported, если устройство не поддерживает управление LBD по SNMP.

getLBDPortLoopStatus

Возвращает состояние порта по LBD (normal, loop).

getISM...

Список функций, возвращающих ту или иную информацию о настройках IGMP Snooping Multicast VLAN. Функции, возвращающие информацию о портах, имеют три формата вывода:

Mask: FF000000
Ports: 1,2,3,4,5,6,7,8
Interval: 1-8

getISMMemberMask

getISMMemberPorts

getISMMemberInterval

Возвращает список member портов в заданном формате.

getISMSourceMask

getISMSourcePorts

getISMSourceInterval

Возвращает список source портов в заданном формате.

getISMTaggedMask

getISMTaggedPorts

getISMTaggedInterval

Возвращает список tagged member портов в заданном формате.

getISMName

Возвращает название ISM VLAN.

getISMReplaceSrcIP

Возвращает настройку replace source ip, если настройка не задана, возвращает 0.0.0.0

getISMRemap

Возвращает настройку remap priority.

getISMReplace

Возвращает настройку replace priority (enabled, disabled).

getRADIUS...

Список функций для получения информации о настроенных RADIUS-серверах.

getRADIUSList

Возвращает список с индексами указанных на устройстве RADIUS-серверов.

getRADIUSIP

Возвращает IP адрес RADIUS сервера с указанным индексом.

print $dlink->getRADIUSIP(1);# 192.168.1.112

getRADIUSAuthPort

Возвращает порт авторизации RADIUS сервера с указанным индексом.

getRADIUSAcctPort

Возвращает порт учета RADIUS сервер с указанным индексом.

getRADIUSTimeOut

Возвращает значение таймаута в секундах для RADIUS сервера с указанным индексом. Даже если устройство не поддерживает различные таймауты для каждого RADIUS сервера отдельно, индекс сервера необходимо указывать.

getRADIUSRetransmit

Возвращает количество повторных попыткок соединения для RADIUS сервера с указанным индексом. Даже если устройство не поддерживает раздельную настройку повторных попыток соединения для каждого RADIUS сервера отдельно, индекс сервера необходимо указывать.

getIMBP...

Список функций для получения информации о настройках IP-MAC-Port binding.

getIMPBPortState

Возвращает настройку IP-MAC-Port Binding для указанного порта (disabled, strict, loose). Если устройство не поддерживает функционал IMPB, будет возвращено not_supported.

getIMPBDHCPSnooping

Возвращает состояние глобальной настройки DHCP Snooping (enabled, disabled). Если устройство не поддерживает функционал IMPB, будет возвращено not_supported.

getIMPBPortZeroIP

Возвращает состояние настройки IMPB Allow Zero IP для указанного порта (enabled, disabled). Если устройство не поддерживает функционал IMPB, будет возвращено not_supported.

getIMPBForwardDHCPPkt

Возвращает состояние настройки IMPB Forward DHCP Packet для указанного порта (enabled, disabled). Если устройство не поддерживает функционал IMPB, будет возвращено not_supported.

getIMPBEntries

Возвращает список с тройками IP-MAC-Port из таблицы валидных связок.

@table = $dlink->getIMPBEntries;
while (@table) {
my $ip = shift @table;
 my $mac = shift @table;
 my $port = shift @table;
 print "$ip - $mac - $port\n"; # 172.17.33.50 - 00-11-22-33-44-55 - 15
}

getIMPBBlockedEntries

Возвращает список с тройками IP-MAC-Port из таблицы заблокированных связок.

@table = $dlink->getIMPBBlockedEntries;
while (@table) {
my $vlan = shift @table;
 my $mac = shift @table;
 my $port = shift @table;
 print "$vlan - $mac - $port\n"; # 33 - 00-11-22-33-44-55 - 15
}

getSNTPState

Возвращает состояние настройки SNTP на устройстве (enabled, disabled).

getSNTPPrimary

Возвращает адрес первичного SNTP сервера.

getSNTPSecondary

Возвращает адрес вторичного SNTP сервера.

getSNTPPollInterval

Возвращает период обновления SNTP в секундах.

getDHCPRelayState

Возвращает состояние настройки DHCP Relay (enabled, disabled).

getDHCPLocalRelayState

Возвращает состояние настройки DHCP Local Relay (enabled, disabled).

getIGMP...

Список функций, возвращающих информацию о настройках IGMP Snooping.

getIGMPQuerierVersion

Задает тип реакции (shutdown, drop) Traffic Control на указанном порту.

setTrafCtrlCountdown

Задает интервал времени в секундах для принятия решения о блокировке на указанном порту.

setTrafCtrlInterval

Задает значение настройки Interval в минутых для указанного порта. Если Traffic Control на порту настроен в режиме Shutdown и шторм на порту за это время не прекратился, порт переходит в состояние shutdown forever и может быть включен обратно только вручную.

setSyslog...

Список функций для управления настройками Syslog на устройстве.

createSyslogHost

Добавляет в список Syslog host с указанными индексом и IP адресом.
Внимание! Индексы не назначаются автоматически!

removeSyslogHost

Удаляет из списка запись о Syslog host с указанным индексом.

setSyslogState

Изменяет глобальную настройку (enabled, disabled) Syslog.

setSyslogIP

Задает IP адрес Syslog хосту с указанным индексом.

setSyslogFacility

Задает Facility (local0, local1, local2, local3, local4, local5, local6, local7) Syslog хосту с указанным индексом.

setSyslogSeverity

Задает минимальную категорию событий (all, warn, info, emergency, alert, critical, error, notice, debug), которые будут отправлены на Syslog сервер с указанным индексом.

setSyslogServerState

Задает состояние (enabled, disabled) Syslog сервера с указанным индексом.

VLAN management

Список функций для работы с IEEE 802.1Q VLAN. Все функции в качестве первого аргумента принимают тег, второй аргумент указан в соответствующих описаниях.

createVLAN

Создает VLAN с указанным тегом и именем.

deleteVLAN

Удаляет VLAN с указанным тегом.

setVLANuntaggedMask

Задает в указанном VLAN битовую маску для untagged портов.
Внимание! Если в маске будут указаны порты, не являющиеся egressed в данном VLAN, они будут добавлены в список Egressed и Untagged.

setVLANtaggedMask

Задает в указанном VLAN tagged порты согласно маске.

setVLANuntaggedPorts

Аналогично функции setVLANuntaggedMask, но в качестве аргумента принимает строку с перечислением портов.

setVLANtaggedPorts

Аналогично функции setVLANtaggedMask, но в качестве аргумента принимает строку с перечислением портов.

addVLANuntaggedPorts

Добавляет untagged порт к указанному VLAN.

addVLANtaggedPorts

Добавляет tagged порт к указанному VLAN.

removeVLANPorts

Удаляет указанные порты из указанного VLAN.

setDHCPRelayState

Изменяет состояние (enabled, disabled) DHCP Relay.

setDHCPLocalRelayState

Изменяет состояние (enabled, disabled) DHCP Local Relay.

setTrafSegMask

Задает указанному порту указанную битовую маску Traffic Segmentation.
Важно! Не работает на DGS-3100 Series!

setTrafSegPorts

Задает указанному порту список портов Traffic Segmentation.
Важно! Не работает на DGS-3100 Series!

ACL management

Список функций для управления Ethernet ACL. Во всех случаях первым аргументом является номер профиля ACL.

removeACLProfile

Удаляет заданный профиль ACL.

deleteACLRule

Удаляет заданное правило в заданном профиле.

createACLProfileEtype

Создает профиль Ethernet ACL с фильтом по Ethertype. Первый аргумент - номер профиля, второй аргумент - имя профиля.

createACLProfileMAC

Создает профиль Ethernet ACL с фильтром по MAC.

createACLProfileMAC($profile_id, $profile_name, $type, $mask, $mask2)

profile_id - номер профиля ACL
profile_name - имя профиля ACL
type - тип фильтрации:

dst - MAC получателя
src - MAC отправителя
srcdst - фильтрация по адресам и отправителя, и получателя

mask - маска для MAC отправителя
mask2 - маска для MAC получателя

addACLRuleEtype

Создает правило в профиле Ethernet ACL с фильрацией по Ethertype:

addACLRuleEtype($profile_id, $access_id, $etype, $action, $port)

profile_id - номер профиля
access_id - номер правила
etype - Ethertype в формате 0x0000
action - действие правила (permit, deny)
port - номер порта.

addACLRuleMAC

Создает правило в профиле Ethernet ACL с фильтрацией по MAC:

permit, deny)
port - номер порта
type - тип фильтрации:

dst - MAC получателя
src - MAC отправителя
srcdst - фильтрация по адресам и отправителя, и получателя

mask - маска для MAC отправителя
mac - MAC отправителя
mask2 - маска для MAC получателя
mac2 - MAC получателя

Все MAC адреса можно писать в любой из нижеуказанных нотаций:

001122334455
00-11-22-33-44-55
00:11:22:33:44:55
0011.2233.4455

setFilterDHCPPortState

Изменяет состояние (enabled, disabled) фильтра DHCP на указанном порту.

setFilterNetbiosPortState

Изменяет состояние фильтра Netbios (enabled, disabled) на указанном порту.

clearARPTable

Очищает таблицу ARP.

setSafeguard...

Список функций для управления Safeguard Engine.

setSafeguardAdminState

Изменяет состояние работы (enabled, disabled) Safeguard Engine.

setSafeguardMode

Изменяет режим работы (strict, fuzzy) Safeguard Engine.

setSafeguardTrap

Изменяет состояние настройки (enabled, disabled) Safeguard Engine SNMP Trap.

setSafeguardFallingThreshold

Multicast Groups/Filters management

Список функций для управления multicast группами и фильтрами. Во всех функциях, которые управляют настройками на портах, номер порта всегда идет первым аргументом.

setMcastFilterStateOnPort

Включает/выключает (enabled, disabled) работу фильтра на порту.
Внимание! Данная функция доступна только для DES-3526!

setMcastAccessOnPort

Задает режим работы фильтров на указанном порту (permit, deny).
Внимание! Из представленных моделей коммутаторов эта функция не поддерживается DES-3028!

createMcastFilter

Создает фильтр для multicast групп.

createMcastFilter($index, $name, $start, $end);

index - индекс фильтра
name - имя фильтра
start - первый адрес диапазона групп
end - последний адрес диапазона групп.

removeMcastFilter

Удаляет фильтр multicast групп.
Внимание! На DES-3526 не работает, т.к. там нет отдельного функционала фильтров. Фильтрация на DES-3526 реализована добавлением/удалением на порт именованного диапазона групп, участвующих в IGMP Snooping (об этом ниже).

createIGMPMcastGroup

Создает запись о диапазоне multicast групп, на которые распространяется функционал IGMP Snooping.

createIGMPMcastGroup($name, $start, $end);

name - имя диапазона
start - первый адрес диапазона
end - последний адрес диапазона.

removeIGMPMcastGroup

Удаляет запись о диапазоне multicast групп, на которые распространяется функционал IGMP Snooping. Ввиду различий в реализации функционала на различных моделях и для унификации кода вызов функции выглядит так:

removeIGMPMcastGroup($name, $start, $end);

name - имя диапазона
start - первый адрес диапазона
end - последний адрес диапазона.

addMcastFiltersOnPortByID

addMcastFiltersOnPortByName

deleteMcastFiltersOnPortByID

deleteMcastFiltersOnPortByName

Функции для добавления/удаления multicast фильтров на портах по номеру/имени фильтра.
Важно! На DES-3526 фильтры не имеют номеров, а на DES-3028 - имен. Остальные модели пользуются и номерами, и именами для multicast фильтров.

getARPAgingTimeInMinutes

Возвращает время жизни записей ARP таблицы в минутах. В качестве побочного эффекта очищает таблицу ARP.

setARPAgingTimeInMinutes

Устанавливает время жизни записей ARP таблицы в минутах.

getSNMPHostList

Возвращает список с параметрами SNMP хостов.

my @result = $dlink->getSNMPHostList;
while (@result) {
my $ip = shift @result;
my $version = shift @result;
my $user = shift @result;
print "$ip - $version - $user\n";
}

172.16.128.5 - V2c - dlread
192.168.1.101 - V1 - dlread

removeSNMPHost

Удаляет SNMP хост по указанному IP адресу.

createSNMPHost

Создает SNMP хост.

createSNMPHost($SNMP_host, $version, $community);

SNMP_host - IP адрес SNMP хоста
version - версия протокола SNMP: v1, v2c, v3
community - community или username для отправки SNMP traps.

getRADIUSKey

Возвращает ключ авторизации на RADIUS сервере с указанным индексом.

createRADIUSHost

Создает запись о RADIUS сервере на устройстве.

createRADIUSHost($id, $ip, $key);

id - индекс сервера
ip - IP адрес сервера
key - ключ авторизации на сервере.

removeRADIUSHost

Удаляет запись о RADIUS сервере с укзанным индексом.

setRADIUS...

Список функций для изменения записей о RADIUS серверах на устройстве. Первым аргументом всегда является индекс записи.

setRADIUSIP

setRADIUSKey

setRADIUSAcctPort

setRADIUSAuthPort

setRADIUSRetransmit

setRADIUSTimeOut

Соответственно назначают записи о сервере с указанным индексом IP адрес, ключ авторизации, порт учета(статистики, accounting), порт авторизации, количество повторных попыток соединения, таймаут на ожидание соединения/ответа.
